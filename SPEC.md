# 1ボタン・リズムチャレンジ 仕様書

## 1. 概要

ブラウザで遊べる、1レーン・1ボタンの短時間リズムゲームを作る。

- PCでは `Space` キー、スマートフォンでは画面タップで操作する
- 30秒で1プレイが終了する
- 音源ファイルは使わず、Web Audio APIで電子音を生成する
- ローカルで開発・確認し、完成後にVercelへ公開する

## 2. 目的

- コーディング開始から約30分で、最初から最後まで遊べる状態にする
- URLを共有するだけで、参加者がインストールなしで遊べるようにする
- 残り時間で演出やゲーム性を追加できる、小さな構成にする

## 3. 対象環境

- macOS上の最新ChromeまたはSafari
- iPhoneおよびAndroidの一般的なモバイルブラウザ
- 縦長・横長のどちらでも遊べるレスポンシブ表示
- キーボードがなくても全機能を操作可能にする

## 4. 初版のスコープ

### 実装するもの

- タイトル画面
- STARTボタン
- 3秒の開始カウントダウン
- 1レーンを流れるノーツ
- Spaceキーまたは画面タップによる入力
- `PERFECT / GOOD / MISS` の判定
- スコア、コンボ、残り時間の表示
- Web Audio APIによるリズム音と入力音
- 30秒後のリザルト画面
- REPLAYボタン

### 初版では実装しないもの

- 市販曲や外部音源の利用
- 複数レーン、長押し、同時押し
- 楽曲・難易度選択
- ランキング、ログイン、データベース
- オンライン対戦
- 細かな音ズレ調整画面
- SNS投稿機能

## 5. ゲームルール

### 基本設定

| 項目 | 初期値 |
| --- | ---: |
| プレイ時間 | 30秒 |
| テンポ | 120 BPM |
| 1拍の長さ | 500ms |
| レーン数 | 1 |
| ノーツ数 | 60個を目安 |
| ノーツ移動時間 | 1,500ms |
| ノーツ間隔 | 250ms、500ms、750msを組み合わせる |

- ノーツは画面上部から判定ラインへ向かって移動する
- ノーツ間隔は `1拍 → 0.5拍 → 0.5拍 → 1.5拍 → 1拍 → 1.5拍 → 1拍 → 1拍` のパターンを繰り返す
- ノーツが判定ラインに重なるタイミングで入力する
- 入力時は、未判定ノーツのうち判定ラインに最も近いものを1つだけ判定する
- 判定可能なノーツがなければ空打ちとし、スコアやコンボは変更しない
- 判定時刻を160ms以上過ぎたノーツは自動的にMISSになる

### 判定と得点

| 判定 | 正解時刻との差 | 得点 | コンボ |
| --- | ---: | ---: | --- |
| PERFECT | 80ms以内 | 1,000点 | +1 |
| GOOD | 160ms以内 | 500点 | +1 |
| MISS | 160ms超過 | 0点 | 0に戻す |

- 初版ではコンボによる得点倍率を付けない
- 判定文字は画面中央付近に短時間表示する
- リザルトでは合計得点、最大コンボ、各判定数を表示する

## 6. 画面仕様

### 6.1 タイトル画面

- ゲームタイトル
- 短い説明：「音に合わせてSpaceまたは画面をタップ」
- STARTボタン
- 音が出ることを示す注意書き

STARTボタンを押した操作を起点にAudioContextを開始し、ブラウザの自動再生制限を回避する。

### 6.2 ゲーム画面

- 上部：スコア、コンボ、残り時間
- 中央：1本のレーン、移動するノーツ、判定文字
- 下部：判定ラインとタップ可能な入力エリア

ゲーム中の入力エリアは画面の広い範囲を対象にし、スマートフォンでも押しやすくする。

### 6.3 リザルト画面

- `FINISH!` 表示
- 合計得点
- 最大コンボ
- PERFECT、GOOD、MISSの回数
- REPLAYボタン

## 7. 音仕様

音源ファイルは同梱せず、Web Audio APIのOscillatorNodeとGainNodeで生成する。

- 拍の音：短いクリック音
- 4拍ごとの小節頭：少し高いアクセント音
- 入力成功音：短く明るい電子音
- MISS時：音を鳴らさない、または低い短音

音量は控えめな初期値にし、急な大音量を避ける。音声処理の基準時刻には `AudioContext.currentTime` を使用する。

## 8. タイミングと描画

- ゲーム進行と判定は、フレーム数ではなく経過時刻を基準にする
- 音のスケジュールと判定にはAudioContextの時刻を使う
- ノーツ描画は `requestAnimationFrame` で更新する
- ノーツ位置は「現在時刻と正解時刻の差」から毎フレーム計算する
- タブが一時的に重くなっても、時間経過そのものがずれにくい構成にする

## 9. 操作仕様

| 操作 | PC | スマートフォン |
| --- | --- | --- |
| ゲーム開始 | STARTをクリック | STARTをタップ |
| ノーツ入力 | Space | ゲーム領域をタップ |
| 再プレイ | REPLAYをクリック | REPLAYをタップ |

- Space入力によるページスクロールをゲーム中だけ抑止する
- キーの長押しによる連続入力は無視し、`event.repeat` が真の場合は判定しない
- タップ入力は `pointerdown` を使用し、マウスとタッチを共通化する

## 10. デザイン方針

- 暗い背景に、明るいネオンカラーのノーツと判定ラインを配置する
- 文字は大きく、離れた位置からでも得点と判定を確認できるようにする
- PERFECTは明るい黄色または水色、GOODは緑、MISSは赤で区別する
- 初版では画像素材を使わず、CSSのみで構成する
- 動きが苦手な利用者向けに、OSの `prefers-reduced-motion` を尊重する

## 11. ファイル構成

外部フレームワークやビルド処理を使わない静的サイトとする。

```text
/
├── index.html
├── style.css
├── game.js
├── SPEC.md
└── vercel.json   # 必要な場合のみ
```

## 12. デプロイ方針

- Mac上でゲームを完成させてからVercelへデプロイする
- 静的ファイルのみのため、サーバー処理や環境変数は使わない
- 公開URLをPCとスマートフォンの両方で確認する
- HTTPS環境で音声、入力、レイアウトが機能することを確認する

## 13. 完了条件

以下をすべて満たせば初版完成とする。

1. STARTを押すと3秒後にゲームが始まる
2. 音とノーツが一定のテンポで進行する
3. Spaceと画面タップの両方で判定できる
4. PERFECT、GOOD、MISSが期待どおりに記録される
5. スコアとコンボが正しく更新される
6. 30秒後に入力を停止し、リザルトが表示される
7. REPLAYで状態を初期化して再度遊べる
8. ブラウザのコンソールに重大なエラーが出ない
9. Vercelの公開URLから遊べる

## 14. 余裕があれば追加する機能

優先度順に追加する。

1. ノーツのパターンを単調な4分音符から変化させる
2. コンボに応じた背景や判定ラインの発光
3. ラスト5秒の演出
4. 効果音・音量のON/OFF
5. ベストスコアをlocalStorageに保存
6. EASY / NORMALの2段階難易度
7. 入力タイミングの傾向（早い・遅い）の表示

## 15. 初版の前提と変更可能な値

実装開始時点では以下を採用する。いずれも `game.js` 冒頭の設定値としてまとめ、短時間で調整できるようにする。

- プレイ時間：30秒
- テンポ：120 BPM
- 判定幅：PERFECT 80ms、GOOD 160ms
- 操作：Spaceまたはタップ
- テーマ：ダーク＋ネオン
- 表示言語：英語中心（START、PERFECT、GOOD、MISS、REPLAY）
